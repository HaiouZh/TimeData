# TimeData CLI 和 Mobile 包审计总结

**审计日期**: 2026-05-17
**审计范围**: packages/cli 和 packages/mobile
**报告文件**: TimeData_CLI_Mobile_Audit_Report_2026-05-17.txt

## 执行摘要

本次审计对 TimeData 项目的 CLI 和 Mobile 包进行了全面的代码审查、安全分析和配置检查。审计发现了 **22 个问题**，其中包括 **1 个严重问题**、**10 个高优先级问题** 和 **11 个中优先级问题**。

### 关键发现

#### 🔴 严重问题（需立即修复）
1. **cleartext traffic 启用** (问题 11)
   - 文件: `packages/mobile/android/app/src/main/AndroidManifest.xml` 第 5 行
   - 风险: 允许明文 HTTP 通信，可能导致中间人攻击和数据泄露
   - 建议: 立即禁用，仅在开发环境允许

#### 🟠 高优先级问题（应在下个版本修复）
1. 敏感信息可能暴露在日志中 (问题 2)
2. 配置文件权限未验证 (问题 3)
3. HTTP 错误处理不完整 (问题 4)
4. 安全配置不够严格 (问题 10)
5. 备份功能启用 (问题 12)
6. 文件访问权限过于宽泛 (问题 13)
7. 签名密钥管理不安全 (问题 16)
8. 发布流程缺少验证 (问题 19)
9. 调试模式可能在生产环境启用 (问题 20)
10. 缺少应用签名验证 (问题 21)

#### 🟡 中优先级问题（应在后续版本改进）
- 参数解析缺乏严格验证
- Bearer Token 格式未验证
- URL 构造存在注入风险
- log 命令缺少必要的验证
- doctor 命令的检查不完整
- 测试缺少边界情况
- 最小 SDK 版本过低
- ProGuard 配置不完整
- 版本号管理不规范
- 构建脚本缺少错误处理
- 缺少崩溃报告机制
